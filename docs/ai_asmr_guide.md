# AI ASMR 功能使用指南

## 功能概述

AI ASMR功能允许AI助手通过工具调用来控制ASMR音频的播放，为用户提供智能化的助眠体验。

## 主要特性

### 1. 音频控制
- **背景音音量**：控制背景环境音的音量
- **语音音量**：控制AI语音的音量
- **触发音音量**：控制ASMR触发音（如采耳、按摩等）的音量

### 2. AI对话
- 支持连续工具调用，AI可以根据用户需求自动执行多个ASMR操作
- 实时显示AI状态和工具执行反馈
- 文本输入框用于与AI交互

### 3. 可用工具

#### ear_cleaning（采耳）
- **参数**：
  - `position`: 位置（left/right/both）
  - `duration`: 时长（秒）
- **示例**：用户说"帮我采耳"，AI会调用此工具

#### head_massage（头部按摩）
- **参数**：
  - `duration`: 时长（秒）
- **示例**：用户说"给我按摩一下头"

#### hair_brushing（梳头）
- **参数**：
  - `duration`: 时长（秒）
- **示例**：用户说"帮我梳梳头发"

## 工作流程

1. **用户输入**：在输入框中输入需求，如"我想放松一下，帮我采采耳"
2. **AI理解**：AI分析用户需求，决定调用哪些工具
3. **工具执行**：AI调用工具，播放对应的ASMR音频
4. **反馈显示**：工具执行完成后，AI收到反馈并继续对话
5. **连续操作**：AI可以根据需要连续调用多个工具

## 音频文件格式

音频文件位于 `res://assets/audio/sleep/` 目录下，命名格式为：
```
{音效ID}_{序号}_{L或R}.mp3
```

例如：
- `1_1_L.mp3` - 采耳音效，序号1，左声道
- `1_1_R.mp3` - 采耳音效，序号1，右声道

## 配置文件

### asmr_tools.json
位于 `config/asmr_tools.json`，包含：
- **tools**: 可用工具的定义（符合OpenAI工具调用格式）
- **audio_mapping**: 工具与音效ID的映射关系

示例：
```json
{
  "tools": [...],
  "audio_mapping": {
    "ear_cleaning": {
      "audio_id": "1",
      "default_duration": 30
    }
  }
}
```

## 参数弱检查机制

当AI提供的参数不合法时，系统会使用默认值：
- **position** 不合法 → 随机选择 left/right/both
- **duration** ≤ 0 → 使用配置文件中的默认时长

这样可以避免因参数错误导致功能中断，提升用户体验。

## 使用示例

### 示例1：简单请求
```
用户: 帮我采耳
AI: 好的，我现在给你采耳
[执行 ear_cleaning 工具]
AI: 我给你采了30秒的耳朵（双耳）
```

### 示例2：连续操作
```
用户: 我想放松一下
AI: 好的，让我帮你放松。先给你按摩一下头部
[执行 head_massage 工具]
AI: 我给你按摩了20秒的头部
AI: 现在给你梳梳头发
[执行 hair_brushing 工具]
AI: 我给你梳了25秒的头发，感觉怎么样？
```

## 技术实现

### 核心脚本
- `scripts/sleep/ai_asmr_controller.gd`: AI ASMR控制器
- `scripts/sleep/sleep_mode.gd`: 睡眠模式主控制器

### 关键功能
1. **流式响应处理**：实时接收AI的文本和工具调用
2. **工具调用解析**：解析AI返回的工具调用增量数据
3. **音频播放控制**：根据时长循环播放音频文件
4. **对话历史管理**：维护对话上下文，支持连续交互

## 注意事项

1. 确保 `config/ai_config.json` 和 `config/api_keys.json` 已正确配置
2. 音频文件需要按照命名规范放置在正确的目录
3. AI模型需要支持工具调用功能（如GPT-4、Claude等）
4. 工具执行期间会阻塞新的用户输入，直到当前工具完成
