# 陪伴模式重新设计

## 设计理念

陪伴模式现在是一个独立的场景，复用主场景的背景和角色素材，但完全禁用交互功能。这样做的好处：

1. **代码简洁**：不需要在主场景中禁用大量组件和脚本
2. **素材复用**：直接使用主场景的背景图片和角色配置
3. **独立管理**：陪伴模式有自己的生命周期和状态管理

## 核心功能

### 1. 场景结构

- **Background (TextureRect)**: 复用主场景的背景系统，显示书房场景
- **Character (TextureButton)**: 复用角色渲染逻辑，但禁用点击交互（mouse_filter = 2）
- **ExitButton**: 退出陪伴模式按钮
- **PositionTimer**: 定时器，控制角色位置变化（默认5分钟）

### 2. 角色行为

- 固定在书房场景
- 每隔5分钟（可配置）随机变化一次位置
- 位置变化时有淡出和淡入动画
- 使用与主场景相同的角色预设配置

### 3. 背景自动更新

- **天气同步**: 从存档读取当前天气设置
- **时间自动更新**: 根据系统时间自动切换时间段
  - 7:00-16:59 = 白天 (day)
  - 17:00-18:59 = 黄昏 (dusk)
  - 19:00-3:59 = 夜晚 (night)
  - 4:00-6:59 = 黄昏 (dusk)
- 每分钟检查一次时间变化，自动更新背景

### 3. 文件说明

#### `scenes/companion_mode.tscn`
陪伴模式场景文件，定义UI布局和节点结构。

#### `scripts/companion/companion_mode.gd`
陪伴模式主控制器：
- 初始化场景和角色
- 管理定时器
- 处理退出逻辑

#### `scripts/companion/companion_character.gd`
陪伴模式角色控制器：
- 复用主场景角色的核心渲染逻辑
- 移除所有交互功能
- 支持随机位置加载

#### `scripts/companion/companion_mode_manager.gd`
陪伴模式数据管理器（保持原有功能）：
- 会话管理
- 统计数据
- 设置保存

## 使用方式

从主场景进入陪伴模式：
```gdscript
get_tree().change_scene_to_file("res://scenes/companion_mode.tscn")
```

退出陪伴模式返回主场景：
```gdscript
get_tree().change_scene_to_file("res://scripts/main.tscn")
```

## 配置选项

在 `companion_mode.gd` 中可以调整：
- `position_change_interval`: 角色位置变化间隔（秒，默认300秒=5分钟）
- 时间更新定时器的 `wait_time`: 检查时间变化的间隔（默认60秒=1分钟）

## 技术细节

### 天气获取
使用 `SaveManager.get_current_weather()` 方法获取当前天气，确保与主场景保持一致。

### 时间自动更新
- 启动时根据系统时间设置初始时间段
- 每分钟检查一次系统时间
- 当时间段发生变化时，自动重新加载场景背景
- 不影响角色位置（角色保持在原位）

### 角色位置变化动画
1. 淡出动画（0.3秒）
2. 等待0.2秒
3. 重新加载角色到随机位置
4. 淡入动画（0.5秒，在 `companion_character.gd` 中）
