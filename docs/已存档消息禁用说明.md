# 已存档消息操作禁用功能

## 功能说明

当用户创建存档点后，所有在存档点之前的消息将被标记为"已存档"状态，并禁用以下操作：
- **重新生成**：禁用
- **撤回消息**：禁用  
- **继续生成**：禁用

只有播放语音功能对已存档的AI消息仍然可用。

## 实现逻辑

### 核心原理

1. **存档点创建时**：`story_dialog_save_manager.gd` 的 `create_checkpoint()` 会清空 `current_node_messages` 数组
2. **消息判断**：通过比较消息索引与 `current_node_messages` 的大小来判断是否已存档
   - `messages` 数组包含所有消息（已存档 + 未存档）
   - `current_node_messages` 只包含当前节点的未存档消息
   - 如果 `message_index < (total_messages - current_message_count)`，则该消息已存档

### 代码修改

#### 1. `story_function_panel.gd`

添加了对话面板引用：
```gdscript
var story_dialog_panel: Control = null  # 对话面板引用

func set_story_dialog_panel(panel: Control):
    """设置对话面板引用"""
    story_dialog_panel = panel
```

实现了真正的存档检测逻辑：
```gdscript
func _is_message_archived(message_index: int) -> bool:
    """判断消息是否已存档"""
    if not story_dialog_panel or not story_dialog_panel.save_manager:
        return false
    
    # 获取当前节点的消息数量
    var current_messages = story_dialog_panel.save_manager.get_current_node_messages()
    var current_message_count = current_messages.size()
    
    # messages 数组的后 current_message_count 条是未存档的
    var total_messages = messages.size()
    var first_unsaved_index = total_messages - current_message_count
    
    return message_index < first_unsaved_index
```

#### 2. `story_dialog_panel.gd`

在创建功能面板时设置引用：
```gdscript
# 创建功能面板
var FunctionPanel = preload("res://scripts/story/story_function_panel.gd")
function_panel = FunctionPanel.new()
function_panel.size_flags_vertical = Control.SIZE_EXPAND_FILL
function_panel.visible = false

# 设置对话面板引用
function_panel.set_story_dialog_panel(self)
```

## 按钮禁用规则

| 操作 | 条件 | 已存档时 |
|------|------|----------|
| 播放语音 | AI消息 | ✅ 可用 |
| 重新生成 | AI消息 + 未存档 + AI未响应 | ❌ 禁用 |
| 撤回消息 | 用户消息 + 未存档 + AI未响应 | ❌ 禁用 |
| 继续生成 | 最新AI消息 + 未存档 + AI未响应 | ❌ 禁用 |

## 防止的问题

1. **数据不一致**：防止修改已存档的消息导致存档点内容与实际对话不符
2. **故事树混乱**：防止已创建子节点的对话被修改，保持故事树结构完整性
3. **用户困惑**：明确告知用户哪些消息可以修改，哪些已经固化

## 测试建议

1. 发送几条消息（用户 + AI）
2. 创建存档点
3. 选中存档点之前的消息，验证"重新生成"、"撤回消息"、"继续生成"按钮是否禁用
4. 选中存档点之后的新消息，验证按钮是否正常启用
5. 验证播放语音功能对已存档的AI消息仍然可用
