# 自定义服装创建指南

本指南将帮助你创建自己的角色服装，包括场景立绘和自定义表情。
> 请注意：可能需要一些脑子

## 目录结构

自定义服装需要放置在用户数据目录中：

```
clothes/
├── configs/
│   └── my_costume.json          # 服装配置文件
└── images/
    └── my_costume/              # 服装图片目录（与配置文件同名）
        ├── base.png             # 聊天时的底图（必需）
        ├── preview.png          # 服装预览图（可选，建议100*100px）
        ├── mood_config.json     # 自定义表情配置（可选）
        ├── moods/                # 自定义表情图片目录（可选）
        │   ├── normal.png
        │   ├── happy.png
        │   └── ...
        ├── livingroom/          # 客厅场景立绘
        │   ├── 1.png
        │   ├── 2.png
        │   └── ...
        ├── bedroom/...          # 卧室场景立绘
        ├── bathroom/            # 浴室场景立绘
        ├── studyroom/           # 书房场景立绘
        ├── kitchen/             # 厨房场景立绘
        ├── shop/                # 超市场景立绘
        ├── entryway/            # 玄关场景立绘
        └── rooftop/             # 天台场景立绘
```
> 提示：一次性可以打包多个服装，ID必须严格对应

> 请确保文件夹和文件命名正确，尤其是单复数

> 服装ID请勿设置为`default`或`preset+数字`这种，会与内置服装冲突
## 第一步：创建服装配置文件

创建一个 JSON 配置文件，例如 `my_costume.json`：
>目录：./configs/<服装ID>.json
```json
{
  "id": "my_costume",  //注意：id必须与文件名一致
  "name": "我的服装",  //服装名
  "description": "这是我自定义的服装", //服装描述，会显示在服装选择页面
  "prompt": "你穿着一件蓝色连衣裙", //服装描述，会包含在AI的提示词中
  "livingroom": [
    {
      "des": "坐在沙发上", //立绘描述（其实并没有实际作用）
      "image": "1.png", //图片名
      "scale": 0.8, //图片缩放比例
      "position": {
        "x": 0.5, //水平位置（0.0=左边，1.0=右边）
        "y": 0.6 //垂直位置（0.0=顶部，1.0=底部）
      }
    },
    {
      "des": "坐在地上", //一个场景可以有多个立绘
      "image": "2.png",
      "scale": 0.8,
      "position": {
        "x": 0.5,
        "y": 0.6
      }
    }
  ],
  "bedroom": [...], //场景都是一样的格式
  "bathroom": [...],
  "studyroom": [...],
  "kitchen": [...],
  "shop": [...],
  "entryway": [...],
  "rooftop": [...]

}
```
**请注意：实际使用时需要删除`//双斜杠及后面注释`，否则json无法被正确解析**

## 第二步：准备图片资源

### 2.1 场景立绘

为每个场景创建目录并放置立绘图片
>目录：./images/<服装ID>/<场景ID>/*

- 图片格式：PNG（推荐使用透明背景）
- 图片尺寸：推荐300px左右
- 图片命名：需要与配置文件中的 `image` 字段相对应

### 2.2 聊天底图

创建 `base.png` 作为聊天时的底图
>目录：./images/<服装ID>/base.png

- 尺寸：推荐1000px左右
- 内容：角色聊天时的形象
- **如果没有自定义表情，则必须是在原底图上重绘**

## 第三步：自定义表情**（可选）**

如果你想为服装添加自定义表情，需要创建表情配置和图片。表情会覆盖在聊天底图上
> 提示：如果你的表情每个都是单独的，而不是使用底图，可以创建一个全透明的底图
### 3.1 创建表情配置文件

创建 `mood_config.json`
>目录：./images/<服装ID>/mood_config.json

>注：其实唯一可修改的只有image字段
```json
{
  "moods": [
    {
      "id": 0, //从0开始，每个表情按顺序给定一个数字
      "name": "平静", //表情名
      "name_en": "calm", //表情的英文名（欸，我需要这个吗…？）
      "image": ["normal.png", "normal_2.png"], //表情的图片集，数量不限
      "color": "#FFFFFF", //表情的颜色，显示在边栏的心情文本的颜色
      "weight": 5 //权重，角色倾向于这个心情的权重
    },
    {
      "id": 1,
      "name": "开心",
      "name_en": "happy",
      "image": ["happy.png", "happy_2.png"],
      "color": "#4DFF4D",
      "weight": 1
    }
  ]
}
```
**请注意：实际使用时需要删除`//双斜杠及后面注释`，否则json无法被正确解析**

### 3.2 准备表情图片

>目录：./images/<服装ID>/moods/*

- 图片格式：PNG（必须使用透明背景）
- 图片尺寸：建议与 base.png 相同
- 命名：与 `mood_config.json` 中的 `image` 字段对应

## 第四步：导入和测试

1. 把这些文件打包成.zip文件
> 注意：是在目录里（能看到configs/和images/的页面）打包，而不是把外层文件夹打包
2. 启动游戏
3. 添加服装
4. 选择你的自定义服装
5. 检查各个场景的显示效果
6. 与角色聊天，测试表情切换

## 常见问题

### Q: 如果不想自定义表情怎么办？

A: 不创建 `mood_config.json` 和 `mood/` 目录即可。系统会自动使用内置表情

### Q: 可以只自定义部分表情吗？

A: 不可以。实在没有了可以复制几个同样的

### Q: 只能在配置文件中调整位置和缩放吗？

A: 又寸，下次一定。。。

### Q: 表情图片没有正确叠加怎么办？

A: 检查以下几点：
- 表情图片尺寸是否与 base.png 一致
- 表情图片是否使用了透明背景
- 表情位置是否与 base.png 中的脸部对齐

### Q: 服装在场景中位置不对怎么办？

A: 调整配置文件中的 `position` 和 `scale` 值：
- `position.x` 和 `position.y` 控制位置（0.0-1.0）
- `scale` 控制大小

## 技术细节

### 表情合成原理

系统会在聊天时将 `base.png` 和表情图片合成为一张图片：

1. 加载 base.png 作为底层
2. 加载对应心情的表情图片
3. 将表情图片居中叠加在 base.png 上
4. 使用 Alpha 混合处理透明度
5. 缓存合成结果以提高性能

### 配置加载优先级

1. 服装配置：`user://clothes/configs/` > `res://config/character_presets/`
2. 表情配置：`user://clothes/configs/mood_config.json` > `res://config/mood_config.json`
3. 图片资源：`user://clothes/images/` > `res://assets/images/character/`

### 性能优化建议

从user://加载图片的效率远低于res://，因此建议：
1. 图片尺寸不要过大（建议 1000px 以内）
2. 使用 PNG 格式并适当压缩
3. 表情图片只包含必要的部分
