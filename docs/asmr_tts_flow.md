# ASMR TTS 流程图

## 整体流程

```
AI流式响应 → 分句 → 入队 → TTS合成 → 等待准备 → 播放 → 出队 → 下一个
                ↓
            结束标记 → 等待结束标记 → 工具调用
```

## 详细流程

### 1. AI流式响应阶段

```
用户发送消息
    ↓
AI开始流式响应
    ↓
接收文本块 → process_text_chunk()
    ↓
添加到缓冲区
    ↓
检测中文标点？
    ├─ 是 → 提取完整句子
    │       ↓
    │   计算句子哈希
    │       ↓
    │   入队（playback_queue.append(hash)）
    │       ↓
    │   请求TTS合成（tts_service.synthesize_speech()）
    │       ↓
    │   继续处理剩余缓冲区
    │
    └─ 否 → 继续等待更多文本
```

### 2. 处理循环（持续运行）

```
_processing_loop() 启动
    ↓
    ┌─────────────────────┐
    │ 队列为空？          │
    │  ├─ 是 → 等待一帧   │
    │  └─ 否 ↓            │
    │                     │
    │ 获取队首元素        │
    │  ↓                  │
    │ 是结束标记？        │
    │  ├─ 是 → 出队       │
    │  │       发送信号   │
    │  │       继续循环   │
    │  └─ 否 ↓            │
    │                     │
    │ 等待音频准备        │
    │  ↓                  │
    │ 检查TTS状态         │
    │  ├─ ready → 继续    │
    │  └─ 否 → 等待0.1秒  │
    │          重新检查   │
    │  ↓                  │
    │ 播放句子            │
    │  ↓                  │
    │ 等待播放完成        │
    │  ↓                  │
    │ 出队                │
    │  ↓                  │
    └─ 继续循环 ──────────┘
```

### 3. 流式响应完成

```
AI流式响应结束
    ↓
on_stream_finished()
    ↓
处理剩余缓冲区文本（如有）
    ↓
添加结束标记到队列
    ↓
playback_queue.append("END_MARKER")
```

### 4. 工具调用流程

```
检测到工具调用
    ↓
wait_for_end_marker()
    ↓
    ┌──────────────────────┐
    │ 队列为空？           │
    │  ├─ 是 → 结束等待    │
    │  └─ 否 ↓             │
    │                      │
    │ 队首是结束标记？     │
    │  ├─ 是 → 等待一帧    │
    │  │       继续检查    │
    │  └─ 否 ↓             │
    │                      │
    │ 正在播放/等待？      │
    │  ├─ 是 → 等待一帧    │
    │  │       继续检查    │
    │  └─ 否 → 结束等待    │
    └──────────────────────┘
    ↓
开始执行工具
    ↓
工具执行完成
    ↓
继续对话（新的流式响应）
```

## 队列状态示例

### 示例1：正常播放

```
初始状态：
playback_queue = []

接收句子1 "你好。"
playback_queue = [hash1]
TTS合成中...

接收句子2 "今天天气不错。"
playback_queue = [hash1, hash2]
TTS合成中...

hash1音频准备完成
开始播放hash1...

hash2音频准备完成
等待hash1播放完成...

hash1播放完成，出队
playback_queue = [hash2]
开始播放hash2...

hash2播放完成，出队
playback_queue = []
等待下一个句子...
```

### 示例2：工具调用

```
接收句子1 "你困了吗？"
playback_queue = [hash1]

接收句子2 "让我帮你轻轻采个耳。"
playback_queue = [hash1, hash2]

流式响应结束，添加结束标记
playback_queue = [hash1, hash2, "END_MARKER"]

hash1播放完成，出队
playback_queue = [hash2, "END_MARKER"]

hash2播放完成，出队
playback_queue = ["END_MARKER"]

遇到结束标记，出队
playback_queue = []
发送信号，开始工具调用

执行工具：ear_cleaning
工具执行完成

继续对话...
```

## 关键点

1. **队列驱动**：所有播放都由队列驱动，不依赖外部状态
2. **结束标记**：使用特殊标记来分隔不同的对话轮次
3. **持续循环**：处理循环在初始化时启动，持续运行
4. **等待机制**：通过检查TTSService的状态来等待音频准备
5. **顺序保证**：严格按照入队顺序播放，不会跳过或乱序
