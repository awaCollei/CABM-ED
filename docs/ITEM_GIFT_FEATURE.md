# 送出物品功能实现说明

## 功能概述
实现了在对话中向雪狐送出物品的功能，包括UI交互、提示词构建和AI响应处理。

## 实现的功能

### 1. UI改造
- **"+"按钮**: 原图片上传按钮改为"+"按钮，点击后显示拓展菜单
- **拓展菜单**: 包含"图片"和"物品"两个选项
- **状态切换**: 
  - 未上传时显示默认图标
  - 上传图片后显示图片上传图标
  - 上传物品后显示物品图标
  - 再次点击取消上传

### 2. 物品选择UI
- 使用通用背包UI（`universal_inventory_ui.tscn`）
- 左侧显示玩家背包
- 中间显示单个格子用于放置要送出的物品
- 底部有"提交"按钮确认选择
- 支持拖拽物品到送出格子

### 3. 提示词系统
**用户提示词格式**:
```
【递给你：{物品名}*{数量}】{用户消息}
```

**系统提示词新增字段**:
```
"item": <int> 是否接受物品：0=拒绝，1=收起来[，2=立即使用][，3=使用全部]
```

**动态选项规则**:
- 基础选项: 0=拒绝，1=收起来
- 食品/消耗品: 额外添加 2=立即使用
- 数量>1的食品/消耗品: 额外添加 3=使用全部

### 4. AI响应处理
根据AI返回的`item`字段值进行不同处理:

- **0 (拒绝)**: 物品放回玩家背包
- **1 (收起来)**: 物品放入雪狐背包
- **2 (立即使用)**: 消耗1个，剩余放入雪狐背包
- **3 (使用全部)**: 全部消耗
- **异常情况**: 放回玩家背包

## 使用流程

1. 用户点击"+"按钮
2. 选择"物品"选项
3. 打开物品选择UI，从背包拖拽物品到送出格子
4. 点击"提交"按钮
5. 按钮图标变为物品图标
6. 输入消息并发送
7. 用户消息前自动添加【递给你：物品名*数量】
8. 系统提示词中包含item字段说明
9. AI返回包含item字段的响应
10. 根据item值处理物品（放回/收起/使用）

## 注意事项

1. 不能同时上传图片和物品
2. 物品类型判断基于`config/items.json`中的`type`字段
3. 雪狐背包满时，剩余物品会放回玩家背包
4. 物品数据在AI响应后才会从临时状态清除
5. 如果AI未返回item字段，默认拒绝物品（放回玩家背包）
