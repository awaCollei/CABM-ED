# AI生成选项功能

## 功能概述

在AI响应结束后，自动生成三个对话选项供用户选择，提升对话体验。

## 功能特点

1. **智能生成**：基于当前对话历史，使用总结AI生成三个符合情境的对话选项
2. **便捷交互**：选项显示在屏幕左侧，点击即可发送
3. **可配置**：通过设置面板的"生成选项"开关控制启用/禁用

## 使用方法

### 启用功能

1. 打开设置面板
2. 切换到"回复设置"选项卡
3. 勾选"生成选项"复选框

### 使用流程

1. 与AI进行对话
2. AI响应完成后，系统开始生成选项（后台进行）
3. 当最后一句话显示完毕，点击"继续"
4. 对话框转换为输入模式时，屏幕右侧中间会显示三个对话选项
5. 点击任意选项，该选项会自动发送给AI
6. 如果手动输入消息，选项会自动消失

## 选项生成逻辑

### 提示词设计

系统提示词要求AI：
- 生成3个选项
- 每个选项是用户可能说的话
- 选项自然、符合情境
- 选项之间有所区别
- 每个选项控制在15字以内

### 对话历史处理

- 只使用最近6条对话（避免上下文过长）
- 解析JSON格式的AI回复，提取msg字段
- 过滤空消息

### 响应解析

- 按行分割响应内容
- 移除可能的编号（1. 2. 3.等）
- 确保至少有3个选项
- 只取前3个选项

## UI设计

### 选项容器

- 位置：屏幕右侧，垂直居中
- 宽度：200px
- 间距：10px
- 距离右边缘：20px

### 选项按钮

- 最小尺寸：200x50
- 文字自动换行
- 半透明蓝色背景
- 淡入动画效果

### 显示时机

- AI响应完成后，选项在后台生成
- 当用户点击"继续"，最后一句话显示完毕
- 对话框转换为输入模式时，选项才会显示
- 这样确保用户看完所有内容后才看到选项

## 注意事项

1. 选项生成使用总结模型的配置（model和base_url）
2. 如果总结模型未配置，功能会静默失败
3. 选项生成失败不会影响正常对话流程
4. 用户手动输入或对话结束时，选项会自动清除
5. 默认情况下，功能是关闭的，需要用户手动启用
