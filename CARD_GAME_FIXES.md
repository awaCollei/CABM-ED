# 卡牌游戏修复说明

## 已完成的修复

### 1. 卡牌点击选择机制
- **点击卡牌**：进入"选择"状态，不再直接打出
- **视觉反馈**：
  - 选中的卡牌会向上移动20像素
  - 边框高亮显示（绿色，4像素宽）
  - 使用Godot内置tooltip显示卡牌技能描述（鼠标悬停时自动显示）
- **再次点击**：取消选择

### 2. 卡牌拖动打出机制
- **拖动检测**：通过gui_input事件监听鼠标拖动
- **打出区域**：在tscn中预先定义的PlayZoneIndicator面板
- **视觉指示器**：
  - 拖动时显示半透明绿色区域框（带边框和提示文字）
  - 拖动中的卡牌显示金色边框和半透明效果
- **交互流程**：
  - 先拖动卡牌到绿色区域打出
  - 无目标卡牌：平移到左侧 → 淡出消失
  - 需要目标卡牌：平移到左侧 → 等待选择目标 → 执行后淡出消失

### 3. 结束回合按钮优化
- **始终显示**：不再根据回合状态隐藏
- **禁用状态**：
  - 非玩家回合时置灰
  - 选择卡牌等待目标时置灰
  - 使用灰色样式和禁用字体颜色

### 4. 游戏平衡调整
- **初始抽牌**：从5张减少到3张
- **回合结束**：保留手牌，不再弃牌
- **每回合抽牌**：仍为2张

## 技术实现细节

### 场景文件修改 (battle_scene.tscn)
添加了PlayZoneIndicator节点：
```gdscript
[node name="PlayZoneIndicator" type="Panel" parent="."]
visible = false
z_index = -1
# 覆盖屏幕上方区域（敌方+玩家区域）
```

### 新增变量
```gdscript
@onready var play_zone_indicator = $PlayZoneIndicator
var _dragging_card: int = -1
```

### 新增函数
- `_setup_play_zone_style()`: 设置打出区域指示器的样式
- `_on_hand_card_drag_start()`: 处理拖动开始，显示区域指示器
- `_on_hand_card_drag_end()`: 处理拖动结束，检测是否在打出区域
- `_play_card_from_drag()`: 从拖动执行卡牌（先打出，再选目标）
- `_play_card_move_to_wait()`: 播放卡牌移动到左侧等待动画
- `_play_card_move_and_disappear()`: 播放卡牌移动并消失动画
- `_play_card_final_disappear()`: 播放卡牌最终消失动画
- `_create_draggable_card_button()`: 创建可拖动的卡牌按钮

### 修改的函数
- `_on_hand_card_clicked()`: 改为选择/取消选择逻辑，移除直接显示描述
- `_create_hand_card_ui()`: 
  - 使用内置tooltip显示技能描述
  - 添加拖动状态的视觉反馈（金色边框+半透明）
  - 移除手动显示描述的代码
- `_refresh_turn_panel()`: 优化结束回合按钮的显示逻辑
- `_execute_current_card()`: 修改为选择目标后执行（费用已在拖动时扣除）
- `_on_end_turn_pressed()`: 移除弃牌逻辑

## 使用说明

1. **选择卡牌**：点击手牌进行选择，卡牌会高亮并向上移动
2. **查看技能**：鼠标悬停在卡牌上会自动显示tooltip（技能描述）
3. **打出卡牌**：将卡牌拖动到上方绿色区域释放
4. **选择目标**：需要目标的卡牌会移动到左侧等待选择，选择目标后执行并消失
5. **结束回合**：点击"结束回合"按钮（选择卡牌时会置灰）

## 关键改进

- ✅ 使用Godot内置tooltip系统，更符合引擎规范
- ✅ 在tscn中预定义打出区域，便于可视化编辑
- ✅ 拖动时有明确的视觉反馈（卡牌半透明+金色边框）
- ✅ 先拖动打出，再选择目标的流程更符合卡牌游戏习惯
- ✅ 费用在拖动打出时扣除，避免重复扣费
